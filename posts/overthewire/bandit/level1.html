OverTheWire Bandit Level 1 Write-up

Updated: 14/04/2024
| 1 min read

In this level we have a file called "-" located inside ~

				
					bandit1@bandit:~$ ls
					-

However if try to cat the file in the terminal we'll see that it's not quite working as expected. 

				
					bandit1@bandit:~$ cat -
					foo
					foo
					bar
					bar
					foobar
					foobar
					^C
tip: you can use ctrl+c to terminate any running command-line process

If we reference the cat man page, we will see that it mentions "With no FILE, or when FILE is -, read standard input". Standard input (or stdin, file handle 0) in this case means whatever input we provide with our keyboard, which results in cat returning whatever we type to it back. To get around this issue we have to specify the directory path of the file like so: 

				
					bandit1@bandit:~$ cat ./-
					rRGizSaX8Mk1RTb1CNQoXTcYZWU6lgzi

Which results in the password for level 2!

note: in this example the relative path to the file was used by starting from our current directory (indicated with a dot), however we could have used the absolute file path too, which starts from the root of the filesystem (/), by executing the following: 
				
					bandit1@bandit:~$ cat /home/bandit1/-
					rRGizSaX8Mk1RTb1CNQoXTcYZWU6lgzi
