OverTheWire Bandit Level 8 Write‑up

Updated: 18/04/2024
| 0.5 min read

For this level we know that the password is stored inside data.txt again, but the hint this time is that it's the only line of text which occurs once. The command that will do the job for us is the uniq command, which reports or omits repeated lines. However, at first glance we can notice that it's not quite working as expected: 

				
					bandit8@bandit:~$ uniq -u data.txt
					kjIuqjobFBhKw9Mmfj2wAnWbXB2VxSfv
					5Y76FifuxKStZi4CVovF2uPhgLrZnLzG
					AiYd84lOOVTA4gqJPX7f6DH8eG3zwq1W
					---snip---
Explanation of the command:
  • the -u flag specifies only unique lines to be printed, which is the password in our case

If you look at the man page of uniq you will notice that it filters only adjacent matching lines from stdin. The lines in data.txt are stored in no particular order. To solve this problem we can simply sort the content of data.txt with the sort command and pipe the output to uniq afterwards like so: 

				
					bandit8@bandit:~$ sort data.txt | uniq -u
					EN632PlfYiZbn3PhVK3XOGSlNInNE00t

Level 8 done!
