OverTheWire Bandit Level 4 Write‑up

Updated: 14/04/2024
| 0.5 min read

If you list the files in the inhere directory, you'll find out we have multiple options for where the password might be: 

					
						bandit4@bandit:~$ ls
						inhere
						bandit4@bandit:~$ cd inhere/
						bandit4@bandit:~/inhere$ ls
						-file00  -file01  -file02  -file03  -file04  -file05  -file06  -file07  -file08  -file09

We can try to cat all of the files together by using the asterisk glob pattern, which matches zero or more characters, thus every filename in our list. However, it's hard to make sense from the output.

That's why we will have to use the strings command, which takes file content as input and prints only the human-readable characters as output. Making the output easy for us to read. 

					
						bandit4@bandit:~/inhere$ strings ./*
						HRrtZ
						MUb4
						eE}:
						lrIWWI6bB37kxfiCQZqUdOIYfr6eEeqR

Much better! Based on the format of the passwords from the previous levels we can point out the password for level 5 to be in the last line. An alternative solution would be to use the file command, that can determine file types, to figure out which one of the available files has ASCII text. 

					
						bandit4@bandit:~/inhere$ file ./*
						./-file00: data
						./-file01: data
						./-file02: data
						./-file03: data
						./-file04: data
						./-file05: data
						./-file06: data
						./-file07: ASCII text
						./-file08: data
						./-file09: data

File07 is the only candidate so that's the file with our previously found password!
